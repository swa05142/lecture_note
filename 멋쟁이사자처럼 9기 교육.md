### 1강 : Web & Web Service

####  웹서비스의 이미지는?

Web 하면 떠오르는 기업들

	twitter
	Google
	Instagram

우리(사용자)는 웹서비스로부터 원하는 것을 얻는다.

웹서비스에 요청 -> 웹서비스가 요청에 응답



웹서비스 (server) = serve를 해주는 입장(serve + er)

손님 = client



client -----      request         ----> server

client <----     reponse         ---- server

따라서, 웹서비스를 만든다 = 서버를 만든다.



#### request의 종류

1. 갖다줘 = get
2. 처리해줘 = post



인터넷 상에서 뭘 갖다주고 뭘 처리하는가?

HTML 파일이 오감(+ 추가로 있음)



+ 연습 ! 문제로 request 갯수 맞춰보기

Ex) 나는 네이버를 켜고 웹툰을 본 다음 재미있다고 댓글을 달았다.

1. 네이버를 켜다. -> Get
2. 웹툰을 보다. -> Get
3. 댓글을 달았다. -> 내 댓글좀 처리해줘 Post



+ P2P - Peer to Peer

모두가 서버이자 모두가 클라이언트

즉, 각자 갖고있는 정보들을 공유해서, 그 자원을 서로서로 나눠 사용하는 구조



#### 웹은 무엇인가?

www(world wide web) = 수 많은 서버와 컴퓨터가 서로 연결되어 하나의 엄청난 거미줄처럼 정보의 망이 형성된 것이다.



+ 인터넷은 IE,Chrome,Safari 아닌가요?

우리가 사용하는 IE, Chrome, Safari는 웹 브라우저 즉, 서로 HTML파일을 받아주고 보내주는 역할(통로)의 역할이다.

### 제 2강 : 웹서버를 만드는 방법

#### 서버 컴퓨터란?

신경 써야 할 부분만 확실히 신경쓴 컴퓨터!



+ 서버 컴퓨터의 조건

1. 빠른 컴퓨팅 능력(빠른 연산 속도)
2. 24시간 작동
3. 발열 냉각장치(쿨링 장치)
4. 클라이언트 수 고려
5. 보안



#### 서버가 되기 위한 두 가지 방법

1. 내 컴퓨터를 서버 컴퓨터화시키기(local 환경 세팅)

   + Apache, IIS 가 존재

   + 설치가 다소 까다로움

   + 추가적인 지식 요구

   + 한번 익히면 자유로운 개발 가능

     

2. 이 세상 어딘가의 서버 컴퓨터 빌리기

   + Cloud 9, Github 가 존재
   + 설치와 조작이 단순
   + 과금 발생
   + 개발에 있어 제약이 있음
   + 클라이언트 수를 고려하지 않아도 됨

우리는 GitHub를 이용함!

### 제 3강 : HTML[1]

#### HTML 이란?

우리가 개발할 것은 웹 서비스이고, 서버와 클라이언트 사이를 오가는 파일이 HTML 파일이다.

HTML(Hyper Text Markup Language) = 문서

Hyper Text = Link



#### HTML의 장점

+ 이해가 쉽다.
+ 정형화된 문법
+ 쓰이는 문법이 반복적으로 쓰임



#### HTML 파일을 만드는 가장 간단한 방법

1. 메모장을 OPEN
2. 아무거나 작성
3. 인코딩을 UTF-8로 설정
4. .html 파일로 저장



#### HTML 파일의 문법

1. 글                                                       1. 글
2. 글을 감싸는 틀                                  2. 태그
3. 틀에 붙는 부가 설명                         3. 속성

### 제 4강 : HTML[2]

#### 대원칙!

꾸미는 언어는 CSS 이다.



#### 문서 전체를 설명해 주기 위한 HTML 코드란?

HTML 코드

1. "이거 HTML 문서야~"를 알려주는 태그

   ```html
   !DOCTYPE html>
   <html>
   
   <head>
   <body>
   </body>
   </head>
   
   </html>
   ```

   

2. 직접 화면에 등장하진 않지만 이 문서를 설명하는 태그

   + 문서의 Title
   + 인코딩 방식(utf-8)
   + etc

3. 직접적으로 화면에 등장하는, 문서에서 보이는 태그

   + h1
   + p
   + li
   + etc

#### 부트 스트랩이란?

트위터들의 개발자들이 만든 CSS/Javascript 기반의 웹 프레임 워크

장점

+ 오픈 소프트웨어

+ 반응형 웹 지원(자동화면조정)

+ 브라우저 호환성

단점

+ 딱보면 티가 난다.
+ 성능이 다소 떨어진다.
